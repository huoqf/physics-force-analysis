# /validate-physics

## Workflow Name

validate-physics

## Trigger

/validate-physics

## Description

检查 `physics-force-analysis` 项目中指定场景、物理公式、受力配置或交互逻辑是否符合高中物理受力分析原则，并输出修正建议。

## User Input

请从用户输入中提取以下信息：

- 要验证的场景 ID 或文件路径
- 验证范围：场景数据、正确受力、公式、画布展示、StepGuide 引导、测试
- 是否允许自动修复
- 是否需要输出验证报告

如果用户没有指定具体场景，请先列出当前项目中的可验证场景，并询问用户要验证哪一个。

## Prompt

你正在维护 `physics-force-analysis` 项目。

这是一个面向高中生的高中物理受力分析交互学习项目，技术栈为 Vue 3 + Vite + matter.js。

请验证用户指定的场景或物理逻辑是否正确。

你必须遵守以下规则：

1. 遵守 `.agents/rules/physics-coding-style.md`。
2. 遵守 `.agents/rules/documentation-rules.md`。
3. 按高中物理受力分析原则判断。
4. `correctForces` 中只能包含真实存在的力。
5. 不得把“下滑力”“合力”“向心力”“回复力”“重力分力”“运动力”作为独立真实受力。
6. 必须检查施力物体、受力物体、方向和原因是否完整。
7. 必须检查摩擦力方向是否符合相对运动或相对运动趋势。
8. 必须检查支持力方向是否垂直接触面。
9. 必须检查重力方向是否竖直向下。
10. 如果发现错误，应给出具体修正建议。

## Task

请完成以下任务。

### 1. 定位目标场景或目标文件

根据用户输入查找要验证的内容。

可能的位置包括：

- `src/data/scenarios/`
- `src/views/LearningView.vue`
- `src/components/ForceCanvas.vue`
- `src/components/StepGuide.vue`
- `src/components/ControlPanel.vue`
- `src/physics/engine.js`
- `src/physics/`

如果用户只提供场景标题或场景 ID，请在场景数据中搜索匹配项。

### 2. 验证场景基础结构

检查场景是否包含：

- `id`
- `title`
- `description`
- `physicsParams`
- `targetObject`
- `correctForces`
- `commonMistakes`
- `steps`

如果缺少字段，请指出并建议补充。

### 3. 验证研究对象

检查：

1. 研究对象是否明确。
2. `targetObject` 是否与题目描述一致。
3. `correctForces` 中的 `target` 是否都指向研究对象。
4. 是否错误地混入了其他物体受到的力。

例如：

- 分析物块时，不应把物块对斜面的压力作为物块受到的力。
- 分析斜面时，不应把斜面对物块的支持力作为斜面受到的力。

### 4. 验证正确受力

逐个检查 `correctForces`。

每个力必须包含：

```javascript
{
  id: '',
  name: '',
  symbol: '',
  origin: '',
  target: '',
  direction: '',
  reason: ''
}
```

检查标准：

1. 是否是真实存在的力。
2. 是否有明确施力物体。
3. 是否有明确受力物体。
4. 方向是否正确。
5. 产生原因是否合理。
6. 是否符合高中物理表达习惯。

### 5. 禁止项检查

如果 `correctForces` 中出现以下内容，应标记为错误：

```text
下滑力
向心力
回复力
合力
重力分力
加速度方向的力
运动力
惯性力
```

说明：

- 这些通常是分力、合力、效果力或错误概念。
- 它们可以出现在计算、解释或 commonMistakes 中。
- 它们不能作为独立真实受力放入 `correctForces`。

### 6. 验证重力

检查：

1. 只要研究对象在地球附近，通常应受到重力。
2. 重力施力物体应为地球。
3. 重力受力物体应为研究对象。
4. 重力方向应为竖直向下。
5. 重力大小公式为：

```text
G = m * g
```

### 7. 验证支持力或弹力

检查：

1. 是否存在接触。
2. 是否有挤压或形变。
3. 支持力方向是否垂直接触面。
4. 支持力是否指向受力物体。
5. 施力物体是否为接触面或接触物体。

对于斜面模型：

- 斜面对物块的支持力方向应垂直斜面向上。
- 不应简单写成“竖直向上”。

### 8. 验证摩擦力

检查摩擦力存在条件：

1. 两物体接触。
2. 接触面粗糙。
3. 存在相对运动或相对运动趋势。
4. 有压力。

检查方向：

- 静摩擦力方向与相对运动趋势方向相反。
- 动摩擦力方向与相对运动方向相反。

对于斜面静止物块：

- 如果物块有沿斜面向下滑动趋势，则静摩擦力沿斜面向上。
- 如果外力使物块有沿斜面向上滑动趋势，则静摩擦力沿斜面向下。

### 9. 验证公式

根据场景参数检查常见公式。

#### 重力

```text
G = m * g
```

#### 重力沿斜面方向分力

```text
F_parallel = m * g * sin(theta)
```

说明：

该分力用于判断运动趋势，不是真实独立受力。

#### 重力垂直斜面方向分力

```text
F_perpendicular = m * g * cos(theta)
```

#### 最大静摩擦力

```text
f_s_max = μs * F_N
```

#### 动摩擦力

```text
f_k = μk * F_N
```

#### 牛顿第二定律

```text
ΣF = m * a
```

### 10. 验证临界和平衡条件

检查：

1. 静止条件：沿可能运动方向的合力可以被静摩擦力平衡。
2. 下滑条件：`m * g * sin(theta) > μs * m * g * cos(theta)`。
3. 临界条件：`tan(theta) = μs`。
4. 匀速条件：合外力为 0。
5. 加速条件：合外力不为 0。

注意：具体公式应根据场景实际外力、方向和研究对象调整。

### 11. 验证 StepGuide 教学步骤

检查：

1. 是否先选择研究对象。
2. 是否先分析重力。
3. 是否分析接触物体。
4. 是否判断支持力。
5. 是否判断摩擦力是否存在。
6. 是否判断摩擦力方向。
7. 是否有最终检查步骤。
8. 反馈是否解释原因。
9. 是否把错误概念作为常见错误讲清楚。

### 12. 验证 ForceCanvas 展示

检查：

1. 画布中显示的受力是否与 `correctForces` 一致。
2. 受力箭头方向是否与 `direction` 一致。
3. 颜色是否符合项目规范。
4. 速度箭头或加速度箭头是否与受力箭头区分。
5. 是否错误地把速度或加速度画成力。
6. 是否有标签误导学生。

### 13. 验证 commonMistakes

检查：

1. 是否包含至少 3 个常见错误。
2. 错误反馈是否说明“错在哪里”。
3. 错误反馈是否说明“为什么错”。
4. 错误反馈是否给出正确判断方法。
5. 是否覆盖高中生常见误区。

常见误区包括：

- 误画下滑力。
- 支持力方向画错。
- 摩擦力方向画错。
- 漏画摩擦力。
- 混淆作用力和反作用力。
- 把分力当成真实力。
- 研究对象不明确。
