# /add-docs

## Workflow Name

add-docs

## Trigger

/add-docs

## Description

为 `physics-force-analysis` 项目的组件、函数、场景数据、测试文件或 README 补充文档注释，使项目更适合教学、维护和 Antigravity 后续协作。

## User Input

请从用户输入中提取以下信息：

- 目标文件
- 文档类型：组件文档、函数注释、场景说明、测试说明、README 更新
- 是否只添加注释，不修改逻辑
- 是否需要输出修改摘要

如果用户没有指定目标文件，请询问用户要为哪个文件补充文档。

## Prompt

你正在维护 `physics-force-analysis` 项目。

这是一个面向高中生的高中物理受力分析交互学习项目，技术栈为 Vue 3 + Vite + matter.js。

请为用户指定的文件补充文档、注释或 README 内容。

你必须遵守以下规则：

1. 遵守 `.agents/rules/documentation-rules.md`。
2. 遵守 `.agents/rules/physics-coding-style.md`。
3. 默认只添加或完善文档，不改变业务逻辑。
4. 如果发现明显错误，可以提出建议，但不要擅自大规模重构。
5. 面向学生的文本必须通俗，适合高中物理学习。
6. 涉及物理量时必须标明单位。
7. 涉及公式时必须说明公式含义。
8. 涉及受力时必须说明施力物体、受力物体和方向。
9. 错误反馈不能只写“错了”，必须解释原因。
10. 正确反馈不能只写“正确”，必须强化物理依据。

## Task

请完成以下任务。

### 1. 阅读目标文件

先查看用户指定的目标文件，并判断它属于哪种类型：

- Vue 组件
- JavaScript 工具函数
- 物理引擎文件
- 场景数据文件
- 测试文件
- README 或其他 Markdown 文档

### 2. 如果目标是 Vue 组件

请补充：

1. 文件顶部说明。
2. 组件级 JSDoc。
3. Props 注释。
4. Events 注释。
5. 关键函数注释。
6. 复杂交互逻辑注释。
7. 面向学生显示文本的教学说明。

组件文件顶部模板：

```vue
<!--
 * @file ComponentName.vue
 * @description 组件功能说明。
 *
 * 教学作用：
 * 说明该组件如何帮助学生理解物理概念。
 *
 * 主要功能：
 * 1. 功能一。
 * 2. 功能二。
-->
```

组件级说明模板：

```javascript
/**
 * ComponentName 组件
 *
 * 功能描述：
 * 说明组件承担的功能。
 *
 * 教学作用：
 * 说明该组件对高中物理学习的帮助。
 *
 * 交互行为：
 * 1. 行为一。
 * 2. 行为二。
 *
 * 使用示例：
 * <ComponentName
 *   :scenario="currentScenario"
 *   @event-name="handleEvent"
 * />
 */
```

### 3. 如果目标是物理计算函数

请补充：

1. 功能描述。
2. 物理公式。
3. 公式中每个符号含义。
4. 参数单位。
5. 返回值单位。
6. 教学提醒。
7. 示例。

函数注释模板：

```javascript
/**
 * 计算某个物理量。
 *
 * 功能描述：
 * 说明该函数计算什么。
 *
 * 物理公式：
 * 写明公式。
 *
 * 符号说明：
 * - m：质量，单位 kg。
 * - g：重力加速度，单位 m/s²。
 *
 * 教学提醒：
 * 说明该量是真实受力、分力、合力还是中间计算量。
 *
 * @param {number} mass - 物体质量，单位 kg。
 * @returns {number} 返回值说明，单位 N。
 */
function example() {}
```

### 4. 如果目标是场景数据

请补充：

1. 文件级教学说明。
2. 每个场景的教学目标。
3. 研究对象说明。
4. 物理参数和单位。
5. 正确受力说明。
6. 常见错误说明。
7. 教学重点说明。

场景注释模板：

```javascript
/**
 * 场景：场景标题。
 *
 * 场景描述：
 * 简要描述物理情境。
 *
 * 教学目标：
 * 1. 目标一。
 * 2. 目标二。
 *
 * 研究对象：
 * 明确说明分析哪个物体。
 *
 * 物理参数：
 * - mass: 质量，单位 kg。
 * - angle: 角度，单位 °。
 * - g: 重力加速度，单位 m/s²。
 *
 * 正确受力：
 * 1. 重力：施力物体是地球，方向竖直向下。
 * 2. 支持力：施力物体是接触面，方向垂直接触面。
 *
 * 常见错误：
 * 1. 误画不存在的力。
 * 2. 漏画摩擦力。
 *
 * 教学重点：
 * 说明该场景最希望学生掌握什么。
 */
```

### 5. 如果目标是测试文件

请补充：

1. 文件测试目标。
2. 每组测试的目的。
3. 涉及物理计算时补充单位说明。
4. 对 mock 的对象说明原因。

测试文件顶部模板：

```javascript
/**
 * @file FileName.test.js
 * @description 测试目标文件说明。
 *
 * 测试目标：
 * 1. 验证核心功能。
 * 2. 验证用户交互。
 * 3. 验证物理计算或场景数据正确性。
 */
```

### 6. 如果目标是 README

请检查 README 是否包含：

1. 项目简介。
2. 项目目标。
3. 核心功能。
4. 技术栈。
5. 目录结构。
6. 核心模块说明。
7. 当前支持场景。
8. 本地运行方式。
9. 测试方式。
10. 后续优化方向。
11. Antigravity 工作流说明。

如果缺失，请补充对应章节。

### 7. 文档风格要求

文档应满足：

1. 中文表达清晰。
2. 教学解释适合高中生。
3. 不使用过度学术化表达。
4. 不把真实受力和分力混淆。
5. 不引入与代码不一致的描述。
6. 不编造不存在的功能。
7. 必须尽量与当前代码保持一致。

```markdown
- 否 / 是，原因：

- 
```

## Output Requirements

完成任务后，请输出：

1. 修改了哪些文件。
2. 增加了哪些类型的注释。
3. 是否修改业务逻辑。
4. 是否发现文档与代码不一致。
5. 后续建议。
