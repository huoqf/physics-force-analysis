# 高中物理受力分析训练器

一个面向高中生的物理学习辅助项目，通过网页交互动画帮助学生理解和掌握高中物理中的受力分析方法。

本项目基于 Vue 3、Vite 和 matter.js 构建，重点服务于高中物理力学学习场景，特别适合用于斜面模型、摩擦力判断、支持力判断、物体运动趋势分析和受力图训练。

## 项目简介

高中物理中的受力分析是力学学习的基础，也是学生最容易出错的部分。本项目希望通过可视化、交互式、分步骤的网页学习方式，帮助学生从“看题目”逐步过渡到“选研究对象、找施力物体、判断力的方向、验证受力是否完整”。

项目当前已经具备斜面模型相关的交互式学习功能，能够展示物体、斜面、受力箭头、运动状态，并配合分步教学引导学生完成受力分析。

## 项目目标

本项目的核心目标是让学生能够：

1. 明确什么是研究对象。
2. 区分真实存在的力和人为分解出来的分力。
3. 正确判断重力、支持力、摩擦力、推力等常见力。
4. 理解摩擦力方向与相对运动趋势之间的关系。
5. 能够通过交互动画建立直观的受力分析思维。
6. 能够在不同物理模型中迁移受力分析方法。

## 核心功能

### 交互式物理场景

项目通过画布展示物体、斜面和受力箭头，学生可以通过点击物体选择研究对象，并观察该物体所受的力。

当前已支持的主要交互包括：

- 点击物体选择研究对象。
- 显示或隐藏受力箭头。
- 切换不同受力分析场景。
- 调整部分物理参数。
- 重置当前场景。
- 全屏查看物理动画区域。

### 分步教学引导

项目内置分步教学组件，引导学生按照受力分析的标准流程进行学习。

标准受力分析流程包括：

1. 选择研究对象。
2. 判断是否受重力。
3. 找出所有接触物体。
4. 判断支持力或弹力。
5. 判断是否存在摩擦力。
6. 判断摩擦力方向。
7. 检查是否多画或漏画力。

### 受力向量可视化

项目使用不同颜色区分不同类型的力，使学生能够直观理解每个力的方向和作用对象。

| 力的类型 | 显示颜色 | 说明 |
| --- | --- | --- |
| 重力 | 红色 | 始终竖直向下 |
| 支持力 / 弹力 | 蓝色 | 垂直接触面并指向受力物体 |
| 摩擦力 | 绿色 | 与相对运动趋势或相对运动方向相反 |
| 推力 / 拉力 | 紫色或橙色 | 根据外力方向显示 |
| 速度方向 | 灰色或辅助色 | 表示物体运动状态，不属于受力 |

### 常见错误反馈

项目支持在学习过程中对常见错误进行提示，例如：

- 把“下滑力”当作一个真实存在的力。
- 把支持力画成竖直向上。
- 忘记分析摩擦力。
- 摩擦力方向判断错误。
- 没有明确研究对象。
- 将重力分力和真实受力混淆。

## 核心模块说明

### ForceCanvas.vue

`ForceCanvas.vue` 是项目中的物理画布组件，负责展示物理场景、绘制受力箭头并处理物体点击交互。

主要职责包括：

- 绘制斜面、物块、支撑面等物理对象。
- 根据当前场景显示正确的受力向量。
- 根据不同力的类型使用不同颜色。
- 支持用户点击画布中的物体。
- 根据学习步骤逐步显示受力。
- 展示物体运动方向或速度方向。

该组件是项目的核心可视化模块，直接影响学生对受力方向和物理关系的理解。

### ControlPanel.vue

`ControlPanel.vue` 是参数控制面板组件，用于调整不同场景中的物理参数。

当前可扩展或支持的参数包括：

| 参数 | 说明 | 单位 |
| --- | --- | --- |
| 质量 | 物体质量 | kg |
| 倾角 | 斜面角度 | ° |
| 摩擦因数 | 接触面粗糙程度 | 无单位 |
| 推力 | 外加推力大小 | N |
| 初速度 | 物体初始速度 | m/s |
| 传送带速度 | 传送带运动速度 | m/s |
| 电场强度 | 电场对带电物体的作用 | N/C |
| 磁感应强度 | 磁场强弱 | T |

该组件适合后续继续扩展为“实验探究模式”，让学生通过改变参数观察受力变化。

### StepGuide.vue

`StepGuide.vue` 是分步教学引导组件，负责引导学生按照正确步骤完成受力分析。

主要功能包括：

- 展示当前分析步骤。
- 提供选择题或判断题形式的交互。
- 根据学生选择给出反馈。
- 控制受力箭头逐步显示。
- 帮助学生纠正常见错误。
- 引导学生完成完整的受力分析流程。

后续建议在该组件中增加“提交答案”和“受力分析验证”功能，用于判断学生最终选择的受力是否完整、是否多画或漏画。

### LearningView.vue

`LearningView.vue` 是主学习页面，负责组织整个学习界面。

当前页面结构大致包括：

- 左侧：场景选择区域。
- 中间：物理动画画布。
- 右侧：教学引导和分析步骤。
- 顶部或局部：场景信息、重置、全屏等操作。

该页面是项目的主容器，负责协调当前场景、当前步骤、已确认受力和用户操作。

### engine.js

`engine.js` 是项目中的物理引擎封装文件，基于 matter.js 实现物理场景的创建、渲染和更新。

主要职责包括：

- 初始化物理引擎。
- 创建物体、斜面、约束等物理对象。
- 处理不同类型的物理模型。
- 管理物体运动状态。
- 为画布组件提供物理状态数据。
- 支持后续扩展更多力学模型。

## 受力分析基本原则

本项目中的受力分析遵循高中物理常用分析方法。

### 第一步：明确研究对象

受力分析必须先确定研究对象。不同研究对象受到的力不同，不能把其他物体受到的力画到当前研究对象上。

例如：

- 分析物块时，只画物块受到的力。
- 分析斜面时，只画斜面受到的力。
- 分析整体时，不画系统内部相互作用力。

### 第二步：先分析场力

高中阶段最常见的场力是重力。

重力的特点：

- 施力物体是地球。
- 受力物体是被研究对象。
- 方向始终竖直向下。
- 大小为：

```text
G = m × g
```

### 第三步：再分析接触力

接触力必须满足“接触”和“发生形变或有相互作用趋势”两个条件。

常见接触力包括：

- 支持力。
- 压力。
- 拉力。
- 推力。
- 摩擦力。
- 绳子张力。
- 弹簧弹力。

### 第四步：判断弹力方向

弹力方向通常垂直接触面，或者沿绳、杆、弹簧方向。

常见判断方法：

- 平面支持力：垂直接触面指向受力物体。
- 绳子拉力：沿绳并指向绳收缩方向。
- 弹簧弹力：沿弹簧轴线方向。
- 杆的作用力：可能沿杆，也可能不沿杆，需要结合题意判断。

### 第五步：判断摩擦力

摩擦力存在的条件：

1. 两个物体相互接触。
2. 接触面粗糙。
3. 两物体之间有相对运动或相对运动趋势。
4. 接触面之间有压力。

摩擦力方向：

- 静摩擦力方向与相对运动趋势方向相反。
- 动摩擦力方向与相对运动方向相反。

### 第六步：检查是否多画或漏画

常见不应画出的力包括：

- 下滑力。
- 向心力。
- 回复力。
- 合力。
- 重力分力。
- 加速度方向上的力。

这些名称往往是效果力、合力或分力，不一定是真实存在的施力物体作用在研究对象上的力。

## 物理公式说明

项目中涉及的基础公式包括：

### 重力

```text
G = m × g
```

其中：

- `G` 表示重力，单位 N。
- `m` 表示质量，单位 kg。
- `g` 表示重力加速度，通常取 9.8 m/s²。

### 重力沿斜面方向的分力

```text
F_parallel = m × g × sinθ
```

该分力用于分析物体沿斜面方向的运动趋势，但不能作为一个独立的真实受力画在受力图中。

### 重力垂直斜面方向的分力

```text
F_perpendicular = m × g × cosθ
```

该分力通常用于计算支持力大小。

### 最大静摩擦力

```text
f_s_max = μ_s × F_N
```

其中：

- `f_s_max` 表示最大静摩擦力。
- `μ_s` 表示静摩擦因数。
- `F_N` 表示支持力。

### 动摩擦力

```text
f_k = μ_k × F_N
```

其中：

- `f_k` 表示动摩擦力。
- `μ_k` 表示动摩擦因数。
- `F_N` 表示支持力。

### 牛顿第二定律

```text
ΣF = m × a
```

其中：

- `ΣF` 表示合外力。
- `m` 表示质量。
- `a` 表示加速度。

```javascript
/**
 * 计算重力沿斜面方向的分力。
 *
 * 公式：
 * F_parallel = m * g * sin(theta)
 *
 * 注意：
 * 该分力用于计算，不是一个独立存在的真实受力。
 *
 * @param {number} mass - 物体质量，单位 kg。
 * @param {number} angle - 斜面倾角，单位 °。
 * @param {number} g - 重力加速度，单位 m/s²。
 * @returns {number} 重力沿斜面方向的分力，单位 N。
 */
function calculateGravityParallel(mass, angle, g = 9.8) {
  const angleRad = angle * Math.PI / 180
  return mass * g * Math.sin(angleRad)
}
```

## 项目定位

本项目不是简单的物理公式计算器，而是一个以“受力分析思维训练”为核心的交互式学习工具。

它的重点不是直接给出答案，而是帮助学生理解：

- 为什么这个物体受到这个力。
- 这个力的施力物体是谁。
- 这个力的方向为什么这样判断。
- 哪些力是真实存在的。
- 哪些只是分力、效果力或合力，不能直接画在受力图中。
